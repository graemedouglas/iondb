# IonDB — Agent Context Map

An embedded database engine for the most constrained environments (2 KB RAM to embedded Linux).

## Workspace structure

```
iondb-core       → Core traits, error types, page format (depends on nothing)
iondb-alloc      → Allocators: static pool, bump, system (depends on iondb-core)
iondb-storage    → Storage engines: B+ tree, hash (depends on iondb-core)
iondb-io         → I/O backends: memory, file, raw flash (depends on iondb-core)
iondb-wal        → Write-ahead log (depends on iondb-core)
iondb-tx         → Transactions, MVCC (depends on iondb-core — cross-cutting)
iondb-buffer     → Buffer pool / page cache (depends on iondb-core)
iondb-query      → LINQ-style query DSL (depends on iondb-core)
iondb-facade     → Facade crate (re-exports everything, published as "iondb")
apps/            → Dogfood apps: sensor-log, edge-config, fleet-telemetry
```

## Dependency rules

1. `iondb-core` depends on **nothing** (except `core`/`alloc`).
2. All impl crates depend **only** on `iondb-core`.
3. **No horizontal deps** between impl crates (no storage→wal, etc.).
4. Cross-cutting integration: `iondb-tx` and `iondb-facade` only.
5. `unsafe` confined to `iondb-alloc` and `iondb-io`.

## Non-negotiable invariants

- **`no_std` by default** — must compile for `thumbv6m-none-eabi`.
- **Zero warnings** — `RUSTFLAGS="-D warnings"` on all builds.
- **No `unwrap()`/`expect()`** in library code — use `iondb_core::Error`.
- **No `Box<dyn Error>`** — use `iondb_core::Error` enum.
- **Endianness-aware** — little-endian default for all on-disk formats.
- **Alignment-safe** — no unaligned reads/writes.

## Error handling

Single `iondb_core::Error` enum. Feature-gated variants compiled out when unused.
No `unwrap()`, no `expect()`, no `Box<dyn Error>` in library code.

## Testing

- Structural tests enforce architecture: `cargo test -p iondb-structural-tests`
- `FailpointIoBackend` (`iondb-io`, `failpoint` feature) for crash simulation
- QEMU targets configured in `.cargo/config.toml`
- Miri for UB detection in unsafe code
- CI feature matrix auto-generated by `scripts/feature-matrix.sh`
